# Phase 1 Development Log

## Why custom chunking?

Although libraries like LangChain provide RecursiveCharacterTextSplitter,
Rishi AI requires chunking strategies aware of scriptures, chapters,
verses and commentaries. Therefore the project uses a strategy-based
chunking architecture rather than relying on a single implementation.

## Why pipelines?

Each subsystem (cleaning, chunking, embeddings) follows a common
Pipeline + Strategy pattern to keep components modular and replaceable.
